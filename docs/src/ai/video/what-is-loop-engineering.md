# 什么是 Loop Engineering？

面试官问你："什么是 Loop Engineering？"如果你回答"不就是 ReAct 循环换了个说法吗？"你就两手一摊，只能无语了。

为什么？

因为 ReAct 只是循环的内部机制——怎么在一轮里推理和行动。Loop Engineering 管的是外层——谁来启动循环、循环跑几轮、什么时候该停。

哈喽大家好，我是二哥呀。今天用 3 分钟，带你搞懂 Loop Engineering，也是我们工程四部曲的终章。

面试官问这道题，考三层：第一，它和提示词工程、上下文工程、Harness Engineering 的递进关系；第二，内循环和外循环的区别；第三，循环失败的典型模式怎么防。

好，接下来给你满分回答，照着背就完事了。

先把四部曲串一遍。提示词工程管"怎么说"，上下文工程管"知道什么"，Harness Engineering 管"能做什么不能做什么"，Loop Engineering 管"循环怎么跑、什么时候停"。四层嵌套，每一层包裹前一层。

这概念怎么火起来的？2026 年 6 月，Claude Code 的负责人 Boris Cherny 说了一句话——"我不再亲自给 Claude 写提示词了。我有循环在运行，它们负责给 Claude 下指令。我的工作是写循环。"同一周，Google Chrome 工程总监 Addy Osmani 发文正式命名了 Loop Engineering。

核心要搞清一件事：分清内循环和外循环。

内循环就是我们之前讲过的 ReAct——推理、行动、观察、重复。这个 Harness 已经帮你搞定了，不需要自己写。

外循环才是你要设计的——什么时候启动 Agent、给它喂什么任务、跑完怎么验证、验证不过怎么重来。全程不需要你坐在那里逐句输入提示词。

外循环要工程化，三个问题必须解决。

第一，终止条件。"让测试通过"是好目标，因为成功可以自动检查。"优化代码"是坏目标，Agent 总能找到可以改进的地方，循环永远停不下来。每个生产循环都需要硬性停止条件——迭代上限、成本预算、或者可验证的完成标准。

第二，验证器分离。写代码的模型不能自己给自己打分。Claude Code 的 /goal 命令就是这么设计的——设定完成目标后，一个独立的评估器在每轮结束时检查目标是否达成，干活的 Agent 不自评。

第三，跨循环状态。循环重启后 Agent 会失忆——上下文窗口是全新的。怎么办？把关键信息写进 CLAUDE.md 和文件系统。Agent 每犯一次错，纠正写进文件，下一轮循环读取，不再犯同样的错。代码仓库不会遗忘。

面试官如果继续追问："Loop Engineering 和 Harness Engineering 到底怎么区分？"

告诉他——Harness 是赛车的引擎和底盘，Loop 是赛事调度。Harness 管的是单次 Agent 运行中的权限、工具、错误恢复。Loop 管的是外层——什么时候发车、跑几圈、什么时候进站加油、什么时候终止比赛。

拿 Claude Code 来说。Harness 确保你输入一条指令后，Agent 能安全地读文件、改代码、跑测试。Loop 确保你睡觉以后，Agent 还能按预设的节奏持续工作——每完成一个任务自动捡起下一个，卡住了自动重启，跑完了自动停。

两者配合，Agent 才能真正实现无人值守的连续工作。

最后一句口诀——提示词管嘴巴，上下文管大脑，Harness 管手脚，Loop 管心跳——四件齐活了，Agent 自己跑。
